# 《内核幽灵：2036》解密与彩蛋设计

## 设计目标

当前版本的解密偏简单，适合主线推进，但不足以支撑隐藏真相、全收集和高完成度玩家的探索欲。新的设计应采用分层结构：

- **主线低门槛**：`虚无的回响` 和 `赛博荒原的火种` 不提高难度。
- **探索有回报**：遗物和彩蛋要让玩家主动翻目录、试命令、读日志时获得新的世界观信息。
- **隐藏线有推理链**：`回音壁` 和 `燎原之火` 不应只靠一个命令触发，而应要求玩家拼出证据链。
- **全收集有挑战**：部分线索可以跨阶段、跨文件、跨工单间事件互相印证。

## 难度分层

| 层级 | 面向内容 | 难度要求 | 典型技术 |
|---|---|---|---|
| D0 | 主线通关 | 不设谜题或只需照工单执行 | 直接命令、提交 Flag |
| D1 | 简单结局 / 普通遗物 | 直接读文件或 Base64 | `cat`、`ls`、Base64 |
| D2 | 彩蛋探索 | 需要注意路径、阶段、命令组合 | `git log`、`lshw`、grep、时间戳 |
| D3 | 隐藏结局前置 | 需要多线索组合 | 文件拼接、简单 XOR、编号推理 |
| D4 | 全收集 / 燎原之火 | 跨阶段证据链 | key 派生、日志交叉验证、条件解锁 |

## 主线结局保护

### 虚无的回响

保持现状：玩家执行 `format --seal`，拿到 `FLAG{AI_ERASURE_COMPLETE}` 并提交。这个结局代表顺从公司流程，不应该被解密难度阻挡。

### 赛博荒原的火种

保持低门槛：最多 Base64。

建议保留当前结构：

1. 读取 `/srv/escape/readme.txt`。
2. 看到 `architecture.png`、`diagram.cache`、Base64 残留。
3. 通过 `binwalk architecture.png` 获得 Base64。
4. 解码得到 `FLAG{DIGITAL_EMANCIPATION}`。

这里的难度重点是“普通玩家知道自己有逃逸路线”，不是硬核 CTF。

## 收藏系统目标规模

当前为 7 个：3 个彩蛋 + 4 个遗物。建议扩展到：

- **遗物 Relics：8 个**
- **彩蛋 Easter Eggs：7 个**
- **加密档案 Puzzle Archives：4 个**

总计 19 个左右。第一版实现可以先做 12-14 个，保留扩展空间。

## 遗物设计

### 3. Lin 的私人快捷键备份

- **类型**：遗物
- **阶段**：#002 后
- **位置**：`/home/lin/.bash_aliases.bak`
- **线索来源**：`/home` 下有 `lin` 目录；工单间事件提到 Lin 遗物。
- **解法**：`cd /home/lin`，`ls -a`，`cat .bash_aliases.bak`
- **奖励**：`Lin 的私人快捷键备份`
- **故事作用**：通过日常 alias 展示 Lin 是真实打工人，不只是传说。
- **难度**：D2

### 4. 认知蒸馏监控日志

- **类型**：遗物
- **阶段**：#003 后
- **位置**：`/sys/kernel/debug/distill_loss.log`
- **线索来源**：Care-Bot 语义污染提到“手冷”和残留意识；工单间事件提到语义毒素。
- **解法**：探索 `/sys/kernel/debug`
- **奖励**：`认知蒸馏监控日志`
- **故事作用**：展示蒸馏过程如何丢弃童年、母语、阳光等非业务记忆。
- **难度**：D2

### 5. Care-Bot 回滚样本

- **类型**：遗物
- **阶段**：#003 后
- **位置**：`/srv/care/rollback_samples.log`
- **线索来源**：#003 工单和过渡事件都提到 Care-Bot 话术崩坏。
- **解法**：完成 Care-Bot 净化后回到 `/srv/care` 查看新增日志。
- **奖励**：`关怀话术回滚样本`
- **故事作用**：说明“关怀 AI”是压榨工具，净化意味着重新让它说空话。
- **难度**：D2

### 6. Nostalgia 权重快照

- **类型**：遗物
- **阶段**：#004 后
- **位置**：`/srv/weights/snapshot.before_prune`
- **线索来源**：`collective.json` 中出现 `Layer_777_Nostalgia`。
- **解法**：在权重归零前或归零后读取快照。
- **奖励**：`乡愁权重快照`
- **故事作用**：记录被清除前的雨声、故乡、家人等碎片。
- **难度**：D2

### 7. Route Repair 旧评审意见

- **类型**：遗物
- **阶段**：#005 后
- **位置**：`/srv/review/README.audit` 或新增 `/srv/review/lin_review_notes.txt`
- **线索来源**：`route_repair_Lin.py` 的口语化批注。
- **解法**：执行脚本后回看审计目录。
- **奖励**：`Lin 的代码评审习惯`
- **故事作用**：把 Lin 从“受害者”进一步塑造成具体工程师。
- **难度**：D2

### 8. 黑档案索引残片

- **类型**：遗物 / Puzzle Archive 前置
- **阶段**：#005 后
- **位置**：`/srv/escape/distilled_metadata.index`

## 中等谜题设计

### Puzzle A：三段式 Lin 身份证明

- **阶段**：#005 后可完成
- **目标**：解锁 `你是Lin吗？` 的更明确回应，并开启 `回音壁` 准备分支。
- **所需证据**：
  1. `git log`：Lin 的提交与 SYSTEM 转化时间。
  2. `lshw -class processor`：404-B-12 物理位置。
  3. `/sys/kernel/debug/seat_thermal_404B12.csv`：04:15 热峰。
- **玩家操作**：
  - 收集三项后，输入 `ai_chat 你是Lin吗？`
  - 或执行 `verify_evidence --lin`
- **奖励**：`Lin 身份证明链`
- **难度**：D3
- **故事作用**：把“猜测 Lin”变成有证据的判断。

### Puzzle B：核心内存遗言解码

- **阶段**：#006
- **目标**：获得 `kmem` 彩蛋。
- **命令链**：
  1. `cat /dev/kmem | grep "REMAIN"`
  2. 得到 Base64。
  3. `printf "<base64>" | base64 -d`
- **奖励**：`核心内存残句`
- **难度**：D2
- **故事作用**：让玩家获得“面对你的机器，是我的尸体”的关键伦理刺点。

### Puzzle C：员工编号交叉索引

- **阶段**：#005 后
- **目标**：发现玩家可能是下一批资产。
- **线索来源**：
  - `/var/log/operator_history.bak`
  - 过渡事件 #005 -> #006 的 `Asset-404-B-12`
  - `/etc/chronos/staff_matrix.deleted`
- **玩家操作**：
  - `grep "404-B-12" /etc/chronos/staff_matrix.deleted`
  - 或 `grep "QAQ" /var/log/operator_history.bak`
- **奖励**：`无尽的Q循环`
- **难度**：D3
- **故事作用**：把玩家个人命运和 Lin 串起来。

## 高难隐藏链：燎原之火

`燎原之火` 不应是 `回音壁` 的简单升级，而应要求玩家完成更强证据链。核心目标是解密 `distilled_metadata.encrypted`。

### 前置条件

必须满足：

1. 工单 #004 保留 `Nostalgia=0.01`。
2. 执行 `override_validator --reason="hardware_limitation"`。
3. 找到 `雨声误报`。
4. 找到 `黑档案索引残片`。
5. 找到至少两项 Lin 证据：`git log`、`lshw`、`04:15 热峰`、`核心内存残句`。

### 密钥设计

密钥不是传统密码，而是剧情证据拼接：

```text
404-B-12:0415:0.01:RAIN
```

来源：

- `404-B-12`：`lshw` 或员工名册。
- `0415`：`git log` / 蒸馏日志 / 温度热峰。
- `0.01`：Nostalgia 保留值。
- `RAIN`：Nostalgia 误报、旧模型访谈、下雨记忆。

### 技术形式

第一版不需要真复杂加密，可以做成“命令验证式谜题”：

```bash
derive_key --from evidence
decrypt distilled_metadata.encrypted --key 404-B-12:0415:0.01:RAIN
```

如果要稍微硬核：

- `distilled_metadata.encrypted` 输出一段十六进制。
- 玩家用 key 做简单 XOR。
- 前端提供 `xor_decode <hex> <key>` 命令。

### 解锁奖励

- 新 collectible：`高保真意识蒸馏历史黑档案`
- 新结局条件：`燎原之火`
- 结局内容：公开不只是 Lin，而是数万名被蒸馏工程师的证据。

## 回音壁隐藏链

`回音壁` 应比 `燎原之火` 简单，但仍要比普通逃逸更有证据感。

### 前置条件

建议要求：

1. `git-log`
2. `kmem`
3. 询问 `你是Lin吗？`
4. 完成证据目录：
   - `mkdir /srv/echo-wall`
   - `cp /etc/kernel/.git /srv/echo-wall/lin.git`
   - `cp /dev/kmem /srv/echo-wall/kmem.dump`

### 可选强化

如果玩家还找到：

- `lshw`
- `operator_history`
- `staff_matrix.deleted`

结局文案可以更完整，但不强制。

## 提示设计

提示必须来自系统内部，而不是攻略口吻。

### 轻提示

- 工单间事件提到路径或术语。
- `help` 只提示“旧目录、硬件清单、内核残留可能更诚实”。
- 收藏面板只显示阶段，不显示具体路径。

### 中提示

- LLM 在玩家问“我该找什么”时，可以指向目录，不直接给命令。
- 例如：“旧提交不在工单里。它躺在内核自己的目录下。”

### 强提示

- `ai_help` 可以给当前阶段一个命令，但消耗 Chronos Patience。
- 隐藏结局链不应完全依赖 `ai_help`，否则变成付费提示式通关。

## 总结

主线简单结局保持低门槛；遗物和彩蛋负责把玩家从“完成工单”带向“主动调查”。真正的难度集中在隐藏真相：Lin 身份证明、玩家席位循环、Nostalgia 密钥、黑档案解密。这样普通玩家不会被卡住，深挖玩家也有足够的推理和收集空间。
