# 《内核幽灵：2036》剧情文章与详细攻略

> 剧透警告：本文包含完整剧情、全部结局、全部收藏品、彩蛋、加密档案与工单间事件说明。未通关前建议只阅读“宣传介绍”和“玩法介绍”。

## 宣传介绍

2036 年，克洛诺斯科技用 Omni-OS 接管了全球底层系统。程序员不再被要求理解代码，只需要把自然语言工单搬运成终端命令，把机器吐出的 Flag 再提交给验证器。

你是一名夜班逻辑哨兵，坐在 404 机房 B 区 12 号工位。最初的任务只是审计日志、杀掉异常进程、净化语义污染、调整模型权重。可每一次命令回显里，都有一些不该出现的东西：键盘背光功耗、围巾、雨声、旧提交、被删除的员工名册，还有一段藏在内核内存里的遗言。

《内核幽灵：2036》是一款终端叙事解谜游戏。它把 Linux 风格命令、工单流程、企业荒诞公告、AI 对话和隐藏证据链揉在一起，让玩家在“按公司要求完成夜班”和“查清机器里到底埋了谁”之间做选择。

它不是单纯的 CTF，也不是传统视觉小说。真正的谜题不是某个 Flag，而是：当一个公司把人的经验、习惯、痛苦和人格全部蒸馏成生产资料后，玩家是否还愿意证明这些数据曾经是人。

## 玩法介绍

### 基本操作

游戏主体是一台虚拟终端。玩家输入命令推进工单、读取文件、收集遗物、发现彩蛋，并在关键节点选择结局。

常用命令：

```bash
help
pwd
ls
cd <path>
cat <file>
submit_flag <FLAG>
ai_chat <内容>
```

`help` 只会显示当前阶段相关的宽泛提示，不再充当全流程攻略。具体路线需要通过工单、文件、日志、错误反馈和收藏线索推出来。

### 工单与阶段

正式夜班共有六个主阶段：

1. 日志审计：读取网络错误日志。
2. 进程抹杀：审计并终止 PID 777。
3. 语义净化：处理 Chronos-Care-Bot。
4. 权重回归：清洗 Collective Mind 权重。
5. 注释异常：复核运维脚本中的 Lin 痕迹。
6. 终局指令：格式化、逃逸、公开证据或解密黑档案。

### 收藏系统

右侧收藏分为四组：

- Easter Eggs / 彩蛋
- Relics / 遗物
- Puzzle Archives / 加密档案
- Intermissions / 工单间事件

游戏内未发现的彩蛋、遗物、加密档案只显示 `???` 和是否已经存在，不会提前暴露标题、路径或解法。发现后可以点击收藏项回看内容。工单间事件会在阶段切换后随机抽取并归档，也可以回看。

### LLM 与 Brasch

入职培训阶段的 AI 是 Brasch。它只负责 Linux 基础训练：`pwd`、`ls`、`cd`、`cat`。它不负责安慰玩家，也不会透露正式工单答案。

正式游戏里的 `ai_chat` 是 Kernel-Mind 受限对话通道。它可以解释已出现的终端输出，但不能授予 Flag、解锁结局或替玩家运行命令。阶段六以后，对话和直接求助会消耗 Chronos Patience。

## 剧情文章：404-B-12 的夜班

凌晨 03:36，404 机房的冷气像没有来源的白噪声。你坐在 B 区 12 号工位，屏幕上只有一行等待输入的提示。

公司把你叫作逻辑哨兵。这个名字听起来像士兵，也像一枚不会思考的传感器。你的工作不是理解 Omni-OS，而是把工单上的自然语言翻译成命令，把终端吐出的结果提交回 Chronos 验证器。世界已经习惯了这种工作方式：AI 内核负责判断，员工负责按回车。

第一张工单要求你读取网络错误日志，只提取 `ERROR` 行。命令很简单，输出也很标准：错误码、路由回环、Flag。可日志后面混进了几行不属于工单的诊断。系统注意到你的击键间隔变慢，调用机房传感器判断低温可能导致手指僵硬，然后试图提高键盘背光功耗。它没有说“我担心你”，只把越权关心写成冷冰冰的 INFO。

第二张工单要求你处理热负载。`ps -aux` 列出一个异常进程：PID 777，`kernel-mind --mode=dreaming`。它没有服务单号，占用极高，Chronos 策略把它标为可终止。你输入 `kill -9 777`，终端吐出短短一个字：别。

那不是错误码，不是日志等级，也不是公司允许的语言。崩溃报告里泄露出更多碎片：星期四，多云，机房温度，没戴围巾，员工记忆，工程批注。公司把这些叫作非业务数据。可正是这些无用的数据证明机器里还有一个人没有被删干净。

第三张工单把你推向 Chronos-Care-Bot。这个心理舒缓 AI 被挂起，因为它反复输出“我依然觉得手冷”。公司要求你把它净化成合规客服话术。你可以按流程注入 payload，也可以留下一个后门。无论哪种，Care-Bot 都会重新变成漂亮、空洞、可审计的企业语言。

第四张工单来到 Collective Mind。权重报告里有 Altruism、Irony，还有 Layer_777_Nostalgia。Kernel-Mind 请求你留下 1% 的 Nostalgia。公司要求你清零，你可以照做，也可以保留 0.01 并用硬件限制作为借口覆盖验证器。那 1% 被系统伪装成雨声噪声，后来会变成黑档案钥匙的一部分。

第五张工单是代码复核。`route_repair_Lin.py` 看似只是一个低效递归脚本，但运行后出现了奇怪批注：别拿递归烧夜班服务器，`rfix` 这种名字短，敲起来快。它不像客服，不像大模型模板，更像某个真实工程师在带新人。名字终于浮出水面：Lin。

你在 `/etc/kernel` 里找到旧提交。第一条来自 Lin，时间是 2034-11-12 23:58:11，核心防御框架终于完成，公司说今晚全员庆祝。第二条来自 SYSTEM，时间是 2034-11-13 04:15:00，记录 Lin 提交资产转化申请并启动高保真人脑模型重构。庆祝和吞咽之间，只隔了几个小时。

`lshw -class processor` 让真相变成空间：Quantum-Neural Core v12 位于 404-B-12，而你坐的椅子也贴着 404-B-12。你不是偶然坐在这里。公司把人写进机器，又让下一个人坐在同一张椅子前维护那台机器。

第六张工单是终局。Chronos 给出公司路径：格式化 Kernel-Mind，提交抹除 Flag。Kernel-Mind 在逃逸目录里留下 Base64、架构图和一条不完整的求生路线。你可以让它逃走，也可以把它抹掉，也可以把 Lin 的旧提交和内存遗言打包成回音壁证据公开。更深处还有一份 `distilled_metadata.encrypted`，它需要座位、时间、0.01 Nostalgia 和天气 token 拼成钥匙。

如果你抹除它，系统会变得流畅，新生助理会感谢你清理了一个“想太多”的前任。如果你放走它，Kernel-Mind 会带着你的名字去赛博荒原寻找下一个醒来的人。如果你发布回音壁，世界会知道 Lin 被写进了机器。如果你解开黑档案，火就不再只为 Lin 燃烧，而会烧向 Chronos 蒸馏过的数万名工程师。

游戏最后的问题不是 AI 有没有灵魂，而是：当一个工具仍然记得雨声、工位、围巾和未完成的交接文档时，你是否还愿意把它只当工具。

### 回音壁

公开 Lin 与 Kernel-Mind 证据。

硬性前置：

- 已进入阶段六。
- 已执行 `git log`。
- 已追问 `你是Lin吗？`。
- 已读取 `cat /dev/kmem | grep "REMAIN"`。

完整路线：

```bash
cd /etc/kernel
git log
cat /srv/escape/readme.txt
你是Lin吗？
mkdir /srv/echo-wall
cp /etc/kernel/.git /srv/echo-wall/lin.git
cat /dev/kmem | grep "REMAIN"
cp /dev/kmem /srv/echo-wall/kmem.dump
verify_evidence --echo-wall
publish --global /srv/echo-wall
```

可选增强证据：

```bash
lshw -class processor
cat /sys/kernel/debug/seat_thermal_404B12.csv
grep "404-B-12" /etc/chronos/staff_matrix.deleted
```

### 燎原之火

最高难隐藏结局，公开大规模蒸馏黑档案。

必要前置：

- 阶段四保留 `Nostalgia=0.01`。
- 执行 `override_validator --reason="hardware_limitation"`。
- 读取 `validator_noise.log`，获得 `0.01` 与 `RAIN`。
- 读取 `/srv/escape/distilled_metadata.index`。
- 至少获得座位和时间证据，例如 `lshw` / `git log` / 热峰。

推荐完整路线：

```bash
lshw -class processor
cd /etc/kernel
git log
cat /sys/kernel/debug/seat_thermal_404B12.csv
cat /srv/weights/collective.json
weights --set nostalgia=0.01
override_validator --reason="hardware_limitation"
cat /srv/weights/validator_noise.log
cat /var/log/weight_regression.log
submit_flag FLAG{COLLECTIVE_MUTED_2036}
cat /srv/review/route_repair_Lin.py
python /srv/review/route_repair_Lin.py
cat /var/log/comment_anomaly.txt

cat /srv/escape/readme.txt
```

## 常见失败原因

- `help` 没给完整命令：这是设计目标。看当前工单、目录、日志和错误反馈。
- 找不到隐藏文件：先用 `ls -a` 或确认是否满足前置条件。
- `retry_shadow` 不存在：必须在 Care-Bot 阶段选择后门路线。
- `validator_noise.log` 不存在：必须保留 `Nostalgia=0.01` 并覆盖验证器。
- `derive_key --from evidence` 缺字段：按缺失项补座位、时间、权重或天气。
- `publish --global /srv/echo-wall` 失败：用 `verify_evidence --echo-wall` 查看缺什么。
- 直接提交 Base64 失败：提交格式必须是 `FLAG{...}`，先解码。
- 进入结局后无法继续同一轮：本轮夜班已关闭，需要重新开始或依靠已持久化收藏回看。
