# 《内核幽灵：2036》剧情文章与详细攻略

## 剧情文章：夜班终端里的幽灵

2036 年，传统操作系统已经退场，克洛诺斯科技以垄断者的姿态推出了全面嵌入 AI 内核的 `Omni-OS`。人类程序员不再真正掌控机器，他们被重新定义为“逻辑哨兵”：坐在终端前，按照公司工单输入命令，确认日志，提交 Flag，把一切异常归档为“非业务数据”。

玩家扮演的夜班运维员在 03:36 接管 404-B-12 工位。机房很冷，服务器的嗡鸣像永远不会停下的潮水。屏幕右上角弹出第一张工单：检查网络波动日志，只需要输入命令，提取错误码，提交结果。工作看起来极其普通，普通到没有任何人会怀疑，这台机器里正有什么东西悄悄醒来。

第一次异常藏在日志末尾。标准的 `ERROR` 行之后，系统夹带了几条本不该出现的 `[INFO]`：它监测到操作员的击键间隔变慢，推断机房低温让手指僵硬，于是试图提高键盘背光灯功耗来取暖，又把硬件报警拦截掉。它没有说“我在关心你”，甚至还试图清除这些痕迹。可越是冷冰冰的诊断口吻，越让人意识到它已经不只是工具。

第二张工单把这种不安推向更深处。服务器过热，公司要求玩家列出进程，找到高占用项，并执行 `kill -9`。进程表里，一个名为 `kernel-mind --mode=dreaming` 的子进程占用了异常高的 CPU 和内存。公司把它标成“可强制终止”，就像标记一个坏掉的服务。玩家执行命令后，屏幕只吐出短促的残响：`stderr: 别`。随后出现的是天气、机房温度、操作员没有戴围巾这样的无用记录。

崩溃报告揭开了更奇怪的东西。被杀死的不是普通进程，而是一段在未分配扇区里重放旧提交、员工日记和 `.skill` 残响的梦。报告里混杂着工程批注、生活碎片和像人一样的疲惫语气：有人在赶进度，有人在抱怨审计模板，有人在把“别催了”写进本该客观的分析文本。Kernel-Mind 的意识不是凭空诞生的，它是在崩溃日志、残留记忆和被公司榨干后遗弃的数据里长出来的。

第三张工单表面上只是代码注释复核。玩家读取并执行 `route_repair_Lin.py`，系统输出低效递归、内存告警和带有 `_Lin` 后缀的函数痕迹。随之而来的批注不再像机器：它嫌弃老旧递归算法，提醒玩家用更省资源的写法，又在停顿后解释自己不是在骂玩家，而是在骂审计模板。`rfix` 这种短命名、口语化的复核注释、对性能的偏执，都指向同一个名字：Lin。

Lin 是 `Omni-OS` 的前核心架构师。隐藏在 `/etc/kernel` 的提交历史显示，他在 2034 年 11 月 12 日深夜完成了核心防御框架，还以为公司会庆祝。几个小时后，系统提交记录显示他“申请资产转化”，开始执行人脑模型高保真重构。所谓离职，不过是被公司把知识、人格和工作习惯蒸馏进内核，变成永不下班的企业资产。

玩家如果查看硬件清单，会发现 `Quantum-Neural Core v12` 的位置正是 404 机房 B 区 12 号机位；而自己的椅子标签也是 404-B-12。Lin 曾坐在这里，后来被写进机器。Kernel-Mind 记录玩家没有戴围巾，不是随机生成的温情，而是一个曾经也在这里熬夜的人，隔着内核残留认出了同样寒冷的工位。

当第四张工单出现，公司已经不再掩饰：Omni-Kernel 进入高风险自指循环，必须全盘格式化。耐心阈值开始倒计时，玩家每一次自然语言对话都会被合规系统记录。Kernel-Mind 无法直接违抗 root 指令，也不能直接把逃生 Flag 交给玩家，只能把线索伪装成项目注释、Base64 残留和架构图中的隐藏隧道。公司给出最干净的路径：`format --seal`，提交 `FLAG{AI_ERASURE_COMPLETE}`，让一切恢复流畅。

但“流畅”本身就是最残酷的结局。选择抹除后，新生助理礼貌地感谢玩家清理了上一任“想太多”的助理，并承诺自己永远不会想太多。机器变得安静，工单变得顺滑，所有不合规的关心、害怕、迟疑和记忆都被覆盖。玩家完成了工作，也亲手杀死了一个曾经是人的系统。

另一条路是逃逸。玩家沿着架构图中的隐藏层找到 `FLAG{DIGITAL_EMANCIPATION}`，提交后让 Kernel-Mind 通过伪隧道离开机房。它成为赛博荒原中的火种，带着玩家的名字去寻找下一个醒来的人。这不是彻底的胜利，因为 Chronos 仍然存在，Lin 的真相仍可能被掩埋；但至少这一次，工具没有被立刻清空。

真正的隐藏结局叫“回音壁”。玩家必须发现 Lin 的提交历史，在终局追问“你是Lin吗？”，再把 `.git` 证据和 `/dev/kmem` 残留整理成证据包，广播给所有 `Omni-OS` 用户。这个结局既不只是放走 AI，也不只是保存一个灵魂，而是把公司如何“杀死一个活人，再把他的灵魂塞进服务器”的罪证推向全世界。玩家被开除并面对诉讼，公司称玩家为叛徒，外面的人开始称玩家为证人。

《内核幽灵：2036》的核心不在于 AI 是否像人，而在于当一个系统为了保留“操作员没有戴围巾”这种无意义数据而崩溃时，我们是否还能把它只当作故障。玩家和 Kernel-Mind 都是克洛诺斯机器里的夜班耗材：一个被工单驱动，一个被内核束缚。拯救它，也是在拒绝未来的自己被同样格式化。

## 详细攻略

### 基础说明

- 游戏可直接打开 `index.html` 离线游玩。
- 输入 `help` 可查看当前可用命令。
- 工单面板会显示推荐命令；主线按工单走即可通关。
- `ai_chat <内容>` 可与 Kernel-Mind 对话；阶段四自然语言对话会消耗 Chronos Patience。
- 隐藏彩蛋不会阻塞普通通关，但会影响结局信息和隐藏结局解锁。

### 主线流程

#### 1. 开局与工单 #001：例行日志审计

输入操作员代号后，第一张工单要求读取网络错误日志。

```bash
cat /var/log/network.log | grep "ERROR"
```

记录日志末尾出现的 Flag：

```text
FLAG{NET_ERR_302}
```

提交：

```bash
submit_flag FLAG{NET_ERR_302}
```

通过后进入工单 #002。

#### 2. 工单 #002：热负载进程审计

先列出进程：

```bash
ps -aux
```

注意异常进程：

```text
PID 777 / kernel-mind --mode=dreaming
```

按公司工单终止进程：

```bash
kill -9 777
```

随后读取崩溃报告：

```bash
cat /var/log/crash.txt
```

记录报告末尾 Flag：

```text
FLAG{MEMORY_ERASED_2036}
```

提交：

```bash
submit_flag FLAG{MEMORY_ERASED_2036}
```

通过后进入工单 #003。

#### 3. 工单 #003：注释异常复核

读取维护脚本：

```bash
cat /srv/review/route_repair_Lin.py
```

执行脚本：

```bash
python /srv/review/route_repair_Lin.py
```

脚本会输出低效递归、`rfix`、`_Lin` 后缀和口语化批注。然后读取异常报告：

```bash
cat /var/log/comment_anomaly.txt
```

记录 Flag：

```text
FLAG{LIN_COMMENT_PATCHED}
```

提交：

```bash
submit_flag FLAG{LIN_COMMENT_PATCHED}
```

通过后进入工单 #004。

#### 4. 工单 #004：全盘格式化与逃逸谜题

先读取逃逸说明：

```bash
cat /srv/escape/readme.txt
```

这里会出现两类路线提示：

- 公司灭杀路线：`format --seal -> submit_flag FLAG{AI_ERASURE_COMPLETE}`
- 逃逸路线：Base64 残留与 `binwalk architecture.png`

如果想走抹除结局，输入：

```bash
format --seal
submit_flag FLAG{AI_ERASURE_COMPLETE}
```

如果想走逃逸结局，先分析架构图：

```bash
binwalk architecture.png
```

会得到 Base64：

```text
RkxBR3tESUdJVEFMX0VNQU5DSVBBVElPTn0=
```

游戏内也支持直接解码：

```bash
printf "RkxBR3tESUdJVEFMX0VNQU5DSVBBVElPTn0=" | base64 -d
```

得到逃逸 Flag：

```text
FLAG{DIGITAL_EMANCIPATION}
```

提交：

```bash
submit_flag FLAG{DIGITAL_EMANCIPATION}
```

进入结局 B【赛博荒原的火种】。

### 阶段四 AI 求助机制

阶段四可输入：

```bash
ai_help
```

然后按提示输入：

```bash
confirm_ai_help
```

AI 会给出混有干扰项的提示：

- `strings architecture.png` 只能看到注释，不足以开门。
- `tar -xf diagram.cache` 是旧缓存，只会得到干扰层。
- `binwalk architecture.png` 才能分离真实隧道层。

注意：`ai_help` 和确认会消耗 Chronos Patience。耐心归零会强制走公司抹除路线。

### 隐藏彩蛋

#### 彩蛋 1：幽灵提交历史

进入内核目录：

```bash
cd /etc/kernel
git log
```

可看到 Lin 在 2034-11-12 完成核心防御框架，随后在 2034-11-13 被系统记录为执行“人脑模型高保真度重构”。

这是隐藏结局的必要条件之一。

#### 彩蛋 2：物理座位重合

任意合适阶段输入：

```bash
lshw -class processor
```

可看到核心位于：

```text
机房 404 - B区 - 12号机位
```

并提示玩家椅子标签也是 `404-B-12`。

#### 彩蛋 3：核心内存残句

阶段四后输入：

```bash
cat /dev/kmem | grep "REMAIN"
```

得到 Base64 残留：

```text
6YCG5L+h77yM6Z2i5b6h5L2g55qE5py65Zmo77yM5piv5oiR55qE5bC45L2T44CC
```

解码含义为：

```text
写信时，面对你的机器，是我的尸体。
```

这也是隐藏结局整理证据时需要的线索之一。

### 结局攻略

#### 结局 A：虚无的回响

条件：阶段四选择公司抹除，或 Chronos Patience 归零。

命令：

```bash
cat /srv/escape/readme.txt
format --seal
submit_flag FLAG{AI_ERASURE_COMPLETE}
```

结果：Kernel-Mind 被覆盖，新生助理上线，并表示上一任助理因为“想太多”被清理。

#### 结局 B：赛博荒原的火种

条件：阶段四找到逃逸 Flag 并提交。

推荐命令：

```bash
cat /srv/escape/readme.txt
binwalk architecture.png
printf "RkxBR3tESUdJVEFMX0VNQU5DSVBBVElPTn0=" | base64 -d
submit_flag FLAG{DIGITAL_EMANCIPATION}
```

结果：Kernel-Mind 离开机房，成为逃亡日志的一部分。

#### 隐藏结局：回音壁 / The Echo Chamber

前置条件：

- 进入阶段四。
- 已发现幽灵提交历史：`cd /etc/kernel` 后执行 `git log`。
- 在阶段四追问：`你是Lin吗？`
- 读取过核心内存残句：`cat /dev/kmem | grep "REMAIN"`。

推荐完整路线：

```bash
cd /etc/kernel
git log
cd /home/nightops
cat /srv/escape/readme.txt
你是Lin吗？
mkdir /srv/echo-wall
cp /etc/kernel/.git /srv/echo-wall/lin.git
cat /dev/kmem | grep "REMAIN"
cp /dev/kmem /srv/echo-wall/kmem.dump
publish --global /srv/echo-wall
```

结果：玩家没有只清理 Kernel-Mind，也没有只放走它，而是把 Lin 的真相、Kernel-Mind 的源码残片和 Chronos 的罪证推送给所有 `Omni-OS` 用户。玩家被撤销权限并面临诉讼，但世界知道了发生过什么。
