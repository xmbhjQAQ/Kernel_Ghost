# PRD: 游戏引导、百科与攻略入口完善

## 目标
完善《内核幽灵：2036》的新手引导、百科内容、收藏提示与站内介绍入口，让零基础和轻基础玩家能理解必要命令与玩法，同时避免无关知识干扰和无确认剧透。

## 范围
- 扩展新手教程，尽量覆盖主线和收藏中会用到的命令类型，而不直接泄露正式工单答案。
- 精简百科：删除当前玩法未使用的知识点，补充实际会用到的命令、管道、隐藏文件、Base64、证据验证、key 派生等内容。
- 面向“基础待建立”和“终端基础已具备”的玩家，在收藏 / Collection 的每个小项目控件右上角添加 `Hint` 按钮，提示该收藏项在哪里找或满足什么前置条件；“可直接参与案件”玩家不显示 Hint，保留完整探索难度。
- 在网站右上角添加书本图标按钮，打开弹窗查看作品宣传介绍、玩法介绍，以及可选择查看具体攻略。
- 具体攻略必须有二次确认，确认前不得直接展示剧透内容。
- 复用《剧情文章与详细攻略.md》中已有的宣传介绍、玩法介绍、主线攻略、全收藏攻略内容，必要时做站内阅读格式改写。

## 不在范围
- 不重写核心剧情、关卡顺序或结局机制。
- 不改变 Flag 校验规则。
- 不把 `help` 变成无确认的全流程剧透攻略。
- 不引入外部依赖或远程内容加载。

## 产出
- 更新 `index.html` 中的新手教程、百科弹窗、收藏 Hint 交互、书本介绍/攻略弹窗。
- 如攻略文案需要同步，更新《剧情文章与详细攻略.md》或从其内容抽取到页面内常量。
- 保留或补充轻量前端静态检查测试。

## 验收标准
- 零基础路径能在教程中学到正式流程会用到的基础命令概念：`help`、`pwd`、`ls`、`ls -a`、`cd`、`cat`、管道、`grep`、`submit_flag`，并理解后续还有阶段专用命令。
- 轻基础路径能跳过完整教程，但仍可打开百科查看实际使用知识，不再看到未使用的哈希、古典密码、RSA 泛知识。
- 收藏项只在 `foundation` 为 `beginner` 或 `terminal` 时显示 `Hint` 按钮；`ready` 路径不显示 Hint。
- 点击 Hint 后给出非剧透或轻剧透的位置提示，例如路径、阶段、前置条件，而不是直接标记为已发现。
- 已发现收藏仍可点击回看，`Hint` 按钮不破坏现有收藏回看行为。
- 书本按钮在页面右上角可见，移动端可用；弹窗默认展示宣传介绍和玩法介绍。
- 用户点击“查看攻略”前必须出现确认；取消后不显示攻略，确认后才展示主线攻略和全收藏攻略。
- 攻略弹窗和百科弹窗可关闭，关闭后回到原游戏输入状态。
- 本地进度恢复、重开、清空重来后，新 UI 状态不造成脚本错误。

## 已确认事实
- 当前新手教程只覆盖 `pwd`、`ls`、`cd tickets`、`cat intro.txt`。
- 当前百科包含“哈希”“古典密码”“RSA”，但当前主线和收藏攻略没有实际使用这些知识。
- 当前收藏列表未发现项只显示 `???`、存在状态和泛提示，未提供逐项 Hint 按钮。
- 当前页面已有百科弹窗和工单间事件弹窗，可复用弹窗样式。
- 《剧情文章与详细攻略.md》已有宣传介绍、玩法介绍、主线攻略、结局攻略、全收藏攻略，可作为书本弹窗内容来源。

## 决策记录
- `Hint` 只给非完全具备全部基础的人准备：`beginner` 和 `terminal` 显示，`ready` 不显示。
