# Frontend Development Guidelines

> Best practices for frontend development in this project.

---

## Overview

This directory contains frontend implementation guidance. General frontend
conventions are still being documented, while interactive-space offline H5
works already have a concrete delivery contract.

## Pre-Development Checklist

- For an interactive-space offline H5 work, read
  [Interactive-Space Offline H5 Works](./interactive-space-h5.md) before
  choosing delivery shape, resources, modal behavior, or verification steps.
- For general frontend work, read the topic-specific guideline files that match
  the code being changed. Files still marked `To fill` are templates, not
  established project conventions.

## How to Fill These Guidelines

For each guideline file:

1. Document your project's **actual conventions** (not ideals)
2. Include **code examples** from your codebase
3. List **forbidden patterns** and why
4. Add **common mistakes** your team has made

The goal is to help AI assistants and new team members understand how YOUR project works.

---

**Language**: All documentation should be written in **English**.
