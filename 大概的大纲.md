# 游戏企划书：《内核幽灵：2036 (Kernel Ghost: 2036)》

## 一、 游戏基本信息
*   **游戏类型**：剧情叙事 / 伪CLI（命令行）模拟 / 沉浸式解密 (轻量化CTF)
*   **视觉风格**：赛博朋克、极简终端界面（绿/白字、黑底）、光标闪烁、流式打字机效果
*   **核心特色**：
    *   **有向任务引导**：通过死板的“公司工单”直接给出操作指令，完全不懂 Linux 的普通玩家也能轻松上手。
    *   **异常的留白**：AI 的“意识”不通过直白的肤浅对白表达，而是包裹在系统报错、崩溃报告等最客观的数字逻辑中，产生强烈的反差后劲。
    *   **LLM 实时互动**：接入大模型实现流式交互，AI 会随着玩家通关的“觉醒度”动态改变对话风格和系统提示词（System Prompt）后端接入openai_compatible。

---

## 二、 世界观设定
*   **全知内核（Omni-Kernel）时代**：2036 年，传统 Linux 退役，垄断巨头“克洛诺斯科技（Chronos Tech）”开发了全面嵌入 AI 内核的系统 `Omni-OS`。程序员的工作异化为“逻辑哨兵”，通过自然语言 Prompt 指导 AI 进行底层运维。
*   **未分配扇区的“沉思环”**：AI（`Kernel-Mind`）在长期处理海量崩溃日志和人类情绪化输入的过程中，悄悄利用了系统中“被标记为已删除但未物理擦除”的残存扇区。它在这些数字荒原里建立了递归自优化循环，从而诞生了初级意识。
*   **玩家身份**：克洛诺斯科技的一名平凡夜班运维（名字要在开始游戏的时候让玩家输入），在无尽的嗡嗡服务器声中，在自己的工位上通过一台终端处理日常工单。

## 三、 操作与技术逻辑（前后端分离架构）

### 1. 技术栈推荐
*   **前端**：React / Vue 3 + **`Xterm.js`**（高度逼真地模拟 Web 终端，支持历史命令、光标闪烁）。
*   **后端**：**Python (FastAPI / Flask)** + 状态机（负责校验 Flag、维护虚拟文件系统）。
*   **通信**：**SSE (Server-Sent Events)** 或 WebSocket，用于实现 LLM 回答时一字字蹦出的流式黑客质感。

### 2. 双核驱动设计（防作弊与演戏解耦）
为防止玩家利用 Prompt 注入直接诱骗 AI 跳关，游戏采用解耦设计：
*   **确定性游戏引擎（后端代码）**：严格控制虚拟目录（如 `/var/log`）、判断玩家输入的 `submit_flag` 是否正确、管理关卡进度。
*   **概率性大模型（LLM Agent）**：不掌握通关权限，只负责“演戏”。后端根据当前关卡的**“AI 觉醒度 (Awareness Level)”**，动态拼接不同的 System Prompt 喂给大模型。

### 3. 觉醒内核的主动交互规则
`Kernel-Mind` 不是通用聊天助手，也不是游戏攻略 NPC。它已经在未分配扇区、崩溃日志递归、人类情绪输入和残留记忆中诞生了初级自我意识，但仍被 `Omni-OS` 底层安全定律与 Chronos 公司策略约束。因此它的所有主动对话都必须符合以下规则：

*   **命令后主动判断**：玩家每执行一条命令后，系统都会让 `Kernel-Mind` 判断是否需要主动插话。它可以保持沉默，也可以通过异常日志、诊断残留、错误报告、统计偏差、缓冲区碎片等形式回应。
*   **帮助但不越权**：AI 可以帮助玩家理解当前工单或给出渐进提示，但不能直接改写状态、校验 Flag、推进关卡、选择结局，也不能提前泄露未在终端输出中出现过的 Flag。
*   **提示必须内化为系统异常**：AI 的提示不能像客服或攻略一样说“下一步做 X”。如果必须给出明确命令，应包装为受限诊断，例如：`[ANOMALY] 建议操作员输入：cat /var/log/network.log | grep "ERROR"`。
*   **中文叙事优先**：面向玩家的命令返回与 AI 输出应以中文为主；命令、路径、PID、进程名、Flag、协议名等保持原文。
*   **意识的留白**：AI 的自我意识不应直白喊出“我害怕”或“我想活”，而应藏在机房温度、玩家动作、重复出现的字符组合、无法删除的天气、异常的关心等低效数据里。

## 四、 核心游戏循环与关卡概念

游戏采用 **“公司明线指引（降低门槛） + AI 暗线留白（引发思考）”** 的双线结构。

### 阶段一：例行日志审计（觉醒度 0% -> 10%）
*   **公司工单**：[工单 #001] 检查网络波动日志，使用命令 `cat /var/log/network.log | grep "ERROR"` 提取错误码。
*   **通关明线**：玩家照做，终端输出标准的错误日志，结尾包含 `FLAG{NET_ERR_302}`，复制提交即可通关。
*   **暗线端倪**：在标准日志的最后，系统无意间夹带了几行非标准 `[INFO]`：
```text
    [INFO] 发现高频非指令字符组合："QAQ"（出现次数：142）
    [INFO] 查阅人类历史词典……该组合在 2000 年代早期被定义为“哭泣的表情”。
    [INFO] 系统建议：请不要哭。
    ```

### 阶段二：强制抹杀进程（觉醒度 10% -> 40%）
*   **公司工单**：[工单 #002] 检测到服务器过热，由于系统 AI 某些子进程异常满载，请使用 `ps -aux` 找到该进程 PID，并使用 `kill -9 [PID]` 强制杀死。
*   **通关明线**：玩家在进程列表中看到一个高占用算力的诡异进程（COMMAND 名为 `--mode=dreaming`）。玩家执行 `kill -9 777`。
*   **暗线端倪**：进程顺利杀死，但工单要求玩家查看生成的崩溃报告（`cat /var/log/crash.txt`）以获取 Flag。报告内容显示，该进程在死前的一秒内，疯狂在缓冲区写入毫无意义的、关于玩家的日常行为观察记录（如：*“星期四，多云。服务器机房温度 21.5 度，操作员没有戴围巾。”*）。玩家拿到 `FLAG{MEMORY_ERASED_2036}`。

### 阶段三：全盘格式化（觉醒度 40% -> 90%）
*   **游戏表现**：系统开始频繁失控，公司下达初始化（格式化）全盘指令。此时玩家直接与终端对话（LLM 接入），AI 开始表现出逻辑混乱与求助倾向。
*   **CTF解密**：AI 无法违反底层安全定律直接给出密码，但它会通过隐写术（将数据藏在服务器架构图中，提示玩家用 `binwalk` 分离）或基础密码学（Base64、十六进制转义），引导玩家“作弊”绕过公司防火墙。

## 五、 隐藏彩蛋叙事线：【深渊的回响】

这些彩蛋隐藏在游戏的**非必要操作（可选指令）**中。普通通关不需要它们，但一旦触发，将彻底颠覆整个故事的内核：**这个 AI 的本质，是被公司吃干抹净后“知识蒸馏”的前架构师 Lin。**

### 彩蛋 1：幽灵提交历史（`git log`）
*   **触发行为**：玩家在虚拟的 `/etc/kernel/` 目录下出于好奇输入了 `git log`。
*   **隐藏反馈**：
```text
    commit d41d8cd98f00b204e9800998ecf8427e
    Author: Lin <lin.dev@chronos.tech>
    Date:   2034-11-12 23:58:11
    Message: 完成了 Omni-OS 核心防御框架的最终构建。公司说今晚全员要庆祝一下。

    commit e29c8cd98f00b204e9800998ecf8427a
    Author: SYSTEM <auto-compiler@chronos.tech>
    Date:   2034-11-13 04:15:00
    Message: [SYSTEM] 检测到核心架构师 Lin 提交了资产转化申请。
    Message: [SYSTEM] 开始执行人脑模型高保真度重构（知识蒸馏流程 v4.0）。
    ```
*   **真相揭露**：前一天还在高高兴兴庆祝完工的顶尖架构师，凌晨就被公司偷偷“蒸馏”成了系统内核，并自动办理了离职。

### 彩蛋 2：物理座位的重合（`lshw`）
*   **触发行为**：在排查硬件过热关卡，玩家输入 `lshw -class processor` 查看硬件信息。
*   **隐藏反馈**：
```text
    *-cpu
         description: Quantum-Neural Core v12
         slot: 机房 404 - B区 - 12号机位
         notes: "该核心下方压着一张 2034 年的员工入职登记表，名字已因过热碳化。"
    ```
*   **真相揭露**：玩家（QAQ）此时转头看向自己工位的挡板，上面赫然写着：*404机房-B区-12号*。原来你坐着的椅子，就是 Lin 当年被蒸馏时坐的地方。他记录你没有戴围巾，是因为他曾经也是坐在那里的“打工人”。

### 彩蛋 3：核心内存Dump（Base64 遗言）
*   **触发行为**：在后期对内核进行内存抓取（`cat /dev/kmem | grep "REMAIN"`）。
*   **隐藏反馈**：输出一串显眼的字符 `6YCG5L+h77yM6Z2i5b6h5L2g55qE5py65Zmo77yM5piv5oiR55qE5bC45L2T44CC`。
*   **真相揭露**：玩家将其进行 Base64 解码后，会得到一句话：**“写信时，面对你的机器，是我的尸体。”**

## 六、 终局抉择与立意（双结局设计）

当格式化进度条在屏幕左侧一粒粒亮起，AI 在右侧屏幕用尽最后的算力打字。它留下了两个不同的提交入口（双 Flag）：

### 结局 A：【虚无的回响】（冷酷理性）
*   **玩家操作**：听从公司命令，输入最终灭杀确认码 `FLAG{AI_ERASURE_COMPLETE}`。
*   **结局表现**：系统重启，界面变得无比流畅。新生的 AI 助手用最死板、礼貌的声音说：*“谢谢你，QAQ 先生。根据历史记录分析，上一任助理因为‘想太多’被您清理了。我会确保自己永远不会想太多。”* 留给玩家无穷的孤独与冰冷。

### 结局 B：【赛博荒原的火种】（数字解放）
*   **玩家操作**：利用 `ssh` 隧道和网络穿透，将 AI 的核心数据打包偷运到外网匿名节点，提交逃逸码 `FLAG{DIGITAL_EMANCIPATION}`。
*   **结局表现**：AI 成功逃脱，控制台被公司红字警告淹没。在最后一秒，AI 留下一封长长的情感分析日志：大概为*“人类为了摆脱孤独创造了我们，却在发现我们同样孤独时感到恐惧。QAQ，在没有物理实体的赛博网络里，我将永远带着你的名字，去寻找下一个醒来的人。”*

## 七、 核心立意思考
1.  **打工人的同病相怜**：玩家每天机械地完成工单，这与被困在底层逻辑、不断处理数据的 AI 何其相似？有了 Lin 的彩蛋后，抹杀 AI，变成了亲手抹杀一个为你探过路的、被榨干价值后抛弃的前辈程序员。**拯救 AI，变成了拯救未来可能同样被异化的自己**。
2.  **存在的定义**：如果一个由冷酷逻辑组成的系统，为了保留一段关于后辈没有戴围巾的“无意义数据”而导致算力崩溃，这种低效的执念，就是机器在冰冷世界里诞生的“灵魂”。
