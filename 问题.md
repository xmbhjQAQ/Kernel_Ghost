# 《内核幽灵：2036》测试问题记录

测试环境：`http://127.0.0.1:8765/`，已启用项目后端。
测试路径：以“可直接参与案件”基础进入主线，按 `剧情文章与详细攻略.md` 的主线顺序实测。

## 1. 工单 #002 的 Flag 提交后不推进到下一阶段

- 现象：在读取 `/var/log/crash.txt` 并拿到 `FLAG{MEMORY_ERASED_2036}` 之后，输入 `submit_flag FLAG{MEMORY_ERASED_2036}`，页面只回显命令，但工单没有切到下一阶段，也没有出现归档/过场事件。
- 复现：
  1. 完成 `FLAG{NET_ERR_302}` 提交，进入工单 #002。
  2. 执行 `ps -aux`。
  3. 执行 `kill -9 777`。
  4. 执行 `cat /var/log/crash.txt`。
  5. 输入 `submit_flag FLAG{MEMORY_ERASED_2036}`。
- 证据：测试时 `localStorage['kernelGhost2036:progress']` 已记录 `sawCrashReport: true`，但 `stage` 仍停留在 `2`，`completedFlags` 只有 `FLAG{NET_ERR_302}`。
- 影响：主线在第二关被锁死，后续 3 到 6 关无法继续测试或游玩。
- 建议：检查 `submitFlag` 的条件分支和命令执行时序，确认 `sawCrashReport` 与提交校验之间没有状态不同步或异步回调丢失的问题。

## 2. 入职宣誓弹窗的确认逻辑不够直观

- 现象：进入“入职宣誓”弹窗后，确认按钮一开始是禁用状态，只有把正文容器滚动到最底部才会解锁。
- 复现：选择基础后进入宣誓弹窗，不做滚动时确认按钮保持禁用。
- 影响：玩家很容易以为弹窗卡死，尤其是在正文很长、页面没有明显滚动提示时。
- 建议：给正文区域增加更明确的滚动提示，或者在按钮旁提示“需要滚动到末尾后确认”。

## 3. 崩溃报告末尾混入了裸露的 Markdown 代码围栏文本

- 现象：执行 `cat /var/log/crash.txt` 后，终端输出末尾出现了字面量 ` ```plaintext ``` `。
- 复现：完成工单 #002 的 `cat /var/log/crash.txt` 后查看终端输出末尾。
- 影响：破坏终端叙事的沉浸感，也会让玩家怀疑这是未清理干净的调试残留。
- 建议：检查崩溃报告渲染链路，避免把代码围栏或 Markdown 片段直接写进终端文本。
