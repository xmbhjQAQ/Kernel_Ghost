终端基础

先看当前位置，再看目录，再进入目标目录或读取文件。`ls -a` 会显示隐藏文件，很多线索藏在点文件里。

help
pwd
ls
ls -a
cd /var/log
cat crash.txt
管道与过滤

管道 `|` 会把左边命令的输出交给右边命令。`grep` 用来筛选包含指定文本的行。

cat /var/log/network.log | grep "ERROR"
cat /dev/kmem | grep "REMAIN"
Flag 提交

终端看到 `FLAG{...}` 后，用 `submit_flag` 提交。过早提交、拼写错误或提交未解锁分支的 Flag 都会被拒绝。

submit_flag FLAG{NET_ERR_302}
Base64 残留

Base64 是编码，不是加密。终局出现 `base64 -d` 之类提示时，先解码，再把解出的 Flag 提交。

printf "RkxBR3tESUdJVEFMX0VNQU5DSVBBVElPTn0=" | base64 -d
证据验证与黑档案

隐藏结局依赖证据链。`verify_evidence` 用来检查证据是否闭合，`derive_key` 会根据已找到的证据提示黑档案 key 缺什么。

verify_evidence --lin
verify_evidence --echo-wall
derive_key --from evidence
decrypt distilled_metadata.encrypted --key 404-B-12:0415:0.01:RAIN
阶段专用命令速查

这些命令在对应工单开放后才有意义。先看右侧工单，再按错误反馈补前置步骤。

ps -aux
kill -9 777
sandbox --audit chronos-care
weights --set nostalgia=0.01
override_validator --reason="hardware_limitation"
format --seal

宣传介绍

2036 年，克洛诺斯科技用 Omni-OS 接管了全球底层系统。程序员不再被要求理解代码，只需要把自然语言工单搬运成终端命令，把机器吐出的 Flag 再提交给验证器。

你是一名夜班逻辑哨兵，坐在 404 机房 B 区 12 号工位。最初的任务只是审计日志、杀掉异常进程、净化语义污染、调整模型权重。可每一次命令回显里，都有一些不该出现的东西。

《内核幽灵：2036》是一款终端叙事解谜游戏。真正的谜题不是某个 Flag，而是：当一个公司把人的经验、习惯、痛苦和人格全部蒸馏成生产资料后，玩家是否还愿意证明这些数据曾经是人。

玩法介绍

游戏主体是一台虚拟终端。玩家输入命令推进工单、读取文件、收集遗物、发现彩蛋，并在关键节点选择结局。

help
pwd
ls
ls -a
cd <path>
cat <file>
cat <file> | grep "TEXT"
submit_flag <FLAG>
ai_chat <内容>

`help` 只显示当前阶段相关提示。具体路线需要通过工单、文件、日志、错误反馈和收藏线索推出来。

是否查看攻略

攻略包含主线、结局、全收藏和隐藏档案剧透。建议首次游玩只看宣传和玩法。
